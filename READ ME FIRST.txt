# Keylogger con Captura de Pantalla, Foto y Audio

## Uso Ético
Este proyecto fue desarrollado con fines educativos para entender cómo funcionan los eventos de entrada del sistema y la interacción con distintos componentes del hardware.

No debe ser utilizado en dispositivos de terceros sin su consentimiento explícito. El uso indebido puede violar leyes de privacidad.

---

## Descripción
Este proyecto implementa un sistema de registro de eventos de entrada del teclado junto con la captura periódica de datos del sistema como pantalla, cámara y audio.

El objetivo es analizar cómo se gestionan estos eventos a nivel de software y cómo se almacenan para su posterior procesamiento.

---

## Requisitos
- Windows 10 o superior
- Python (para versión de desarrollo)
- Permisos de usuario para acceso a hardware (cámara/micrófono si se utilizan)

---

## Uso
Ejecutar el programa en un entorno propio y controlado para observar el comportamiento de captura de eventos del sistema.

El programa se ejecuta de forma local y almacena los datos generados en una carpeta de salida para su posterior análisis.

---

## Funcionalidades
- **Registro de teclado**: Captura eventos de entrada del teclado y los almacena en un archivo de texto.
- **Captura de pantalla**: Realiza capturas de pantalla a intervalos definidos.
- **Captura de imagen**: Obtiene imágenes desde la cámara del sistema.
- **Captura de audio**: Registra fragmentos de audio utilizando el micrófono.

---

## Archivos Generados
Los archivos generados se almacenan en una carpeta local de salida (`output/`) que contiene:

- `keylog.txt`: Registro de eventos de teclado.
- `clipboard.txt`: Contenido del portapapeles.
- `photo.jpg`: Imagen capturada desde la cámara.
- `screenshots/`: Carpeta con capturas de pantalla.
- `audios/`: Carpeta con grabaciones de audio.

---

## Notas
- El funcionamiento de la captura de cámara y audio depende de la disponibilidad de los dispositivos en el sistema.
- Si algún dispositivo no está disponible, el programa continúa ejecutándose sin esa funcionalidad.
- Este proyecto está orientado al análisis técnico y no incluye mecanismos de ocultamiento ni transmisión de datos.

---

## Conceptos Aprendidos
- Captura de eventos de teclado en Python
- Manejo de archivos
- Interacción con hardware (cámara, micrófono)
- Automatización de tareas periódicas

---

**Nota:** Este proyecto tiene un propósito exclusivamente educativo, enfocado en comprender la estructura y funcionamiento de este tipo de sistemas a nivel técnico.
